Справочник «Правила конвертации свойств»

Справочник «Правила конвертации свойств» подчинен справочнику «Правила конвертации объектов». Каждый элемент справочника описывет правило формирования образа свойства объекта-приемника в файле. В качестве источника данных может выступать свойство объекта-источника или произвольные данные. Таким образом, основными реквизитами правила конвертации свойств, являются реквизиты Свойство-источник и Свойство-приемник, ссылающиеся на элементы справочника "Свойства объектов" объектов источника и приемника соответственно. Если свойство-приемник имеет ссылочный (агрегатный) тип, то следует указать правило конвертации объектов. Например, при конвертации реквизита Покупатель, документа РасходнаяНакладная, указывается правило конвертации для объектов типа справочник "Контрагенты". Если правило конвертации объектов свойства не указывать, например, в случае, когда свойство-источник имеет составной тип, то, в процессе выгрузки данных, для этого свойства будет предпринята попытка автоматического подбора правила конвертации в соответствии с типом значения свойства-источника. При этом, если подходящих правил конвертации для данного типа источника, существует несколько, то выбрано будет то правило, которое имеет наибольшее значение реквизита Порядок, т.е. правило расположенное ниже в дереве редактирования правил. 

Установка флажка "Синхронизировать объекты по данному свойству при загрузке" - указывает на то, что свойство является синхронизирующим - то есть по данному свойству, при загрузке данных, будет выполнятся поиск существующих объектов информационной базы. Установка флажка "Получить значение из входящих данных" означает то, что конвертируемое значение будет получено не из объекта-источника, а из параметра ВходящиеДанные (данный параметр следует инициализировать в обработчиках событий правила конвертации объектов).

На закладке "Дополнительно", в форме элемента, правило конвертации свойств может быть отключено, посредством установки соответствующего флажка. Данная возможность удобна при отладке правил обмена. Установка флажка "Не замещать значение данного свойства у существующих объектов ИБ" означает то, что для существующих объектов информационной базы, т.е.идентифицированных при загрузке, данное свойство переустанавливаться не будет. В таблице "Запрашиваемые данные" отображаются правила конвертации свойств, у которых установлен флажок "Получить значение из входящих данных" для выбранного правила конвертации объектов свойств. Кнопка "Печать структуры запрашиваемых данных" позволяет автоматически сформировать фрагмент алгоритма, необходимый для инициализации ИсходящихДанных, передаваемых правилу конвертации объектов.
